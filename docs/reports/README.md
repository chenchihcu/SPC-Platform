# Reports Folder Notes

`docs/reports/` 內多數文件是「當時版本」的驗證或修復快照，可能包含後續已被取代的流程文案（例如舊輸出契約、雙模板匯出、UI 路徑之 HTML 報告等）。

使用建議：
- 查「目前有效契約」：優先看 `README.md`（repo root）、`docs/specs/*.md`、`docs/decision-log.md`
- 查「歷史證據」：再回看 `docs/reports/*.md` 對應日期與上下文

原則：
- 報告快照盡量保留原貌，不直接改寫歷史結論
- 若快照內容可能與現況衝突，應補上 superseded 註記，而非覆蓋原始紀錄
