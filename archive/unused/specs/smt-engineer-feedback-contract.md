# SMT Engineer Feedback Contract

本文件定義圖表、統計摘要與報告產出對 SMT 工程師的價值契約。Self-heal 與 UI 變更時須維持以下要點。

## 1. 產出必須回答的三個問題

- **現在有什麼異常？**  
  圖表與摘要應能指出：超出規格、失控點、能力不足、分布異常等。
- **異常發生在哪裡？**  
  能對應到產線可辨識的維度：線別、機台、元件（RefDes / PartType）、區域或批次（BoardNo / Lot）。
- **下一步該檢查什麼？**  
  報告與圖表說明應引導工程師到可執行的檢查（例如某 RefDes 的 Volume 偏高、某批次 Cpk 偏低需調機）。

## 2. 圖表與摘要

- **計數一致**：匯入筆數、分析摘要筆數、圖表使用的資料筆數、報告中的筆數應一致（或明確標示過濾條件與排除原因）。
- **異常語意**：使用 status-pending / status-ok / status-warning / status-error，不以單一紅色代表所有錯誤。
- **不靜默丟棄**：若因規格缺失、欄位缺失或過濾導致部分資料未納入分析，應在摘要或圖表說明中標示（例如「部分 RefDes 無規格，未納入能力分析」）。

## 3. 不確定或無法分類的結果

- 無法明確歸類的發現應標示為 **UNKNOWN** 或 **VERIFY**，並簡述原因（例如「規格未設定，僅顯示分布」）。
- 報告中若某圖表無法產出，應顯示「（無資料或圖表無法產出）」或等效說明，不留下空白或誤導性標題。

## 4. 報告產物

- 報告應包含：資料層級診斷（量測/座標/關聯成功率）、統計環境前提（選定特徵、工單、管制規格）、圖表與簡要結論。
- 圖表匯出失敗時，報告內該區塊應有明確文字說明，不隱藏失敗。

## 5. 驗收檢查（Self-heal / 手動驗證）

- 完成路徑：匯入 → 分析 → 圖表檢視 → 報告匯出，可重現且不中斷。
- 計數在匯入總數、分析摘要、圖表與報告間一致（或在報告中說明差異）。
- 品質閘門未設定或工具不可用時，明確標示為「not configured」或「not available」，不靜默略過。
